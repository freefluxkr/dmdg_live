온디바이스 데이터 보존율

비동기 생산자-소비자 파이프라인

'당목담글'은 마이크와 센서에서 들어오는 대량의 데이터를 병목 없이 처리하기 위해 큐(Queue) 기반의 비동기 아키텍처를 채택했습니다. 생산자가 데이터를 수집하는 동안 소비자(Gemini API)는 최적의 시점에 결과를 생성합니다.

1

Producer (생산자)

Audio/Camera 데이터 실시간 캡처

2

Async Queue (버퍼링)

데이터 흐름 제어 및 병목 방지

3

Consumer (소비자)

Gemini Live API 분석 및 결과 출력

ASYNC

Audio Stream Logic

* PCM 데이터가 오차 범위 내에서 안정적으로 전송되는 추이를 나타냅니다.

정교한 데이터 모델링 (Dexie.js)

IndexedDB를 SQL처럼 다루는 Dexie.js를 사용하여, 페르소나 설정부터 마중물, 릴레이 데이터까지 기기 내에서 완벽한 관계형 구조로 관리합니다.

Schema Complexity vs Speed

테이블 구조가 복잡해지더라도 최적화된 인덱싱을 통해 10ms 미만의 쿼리 속도를 유지합니다.

Persona Trait Distribution

AI 페르소나의 감성 지표(온도, 다정함, 명확성 등)를 분석한 결과입니다.

감성 인지 엔진의 핵심 지표

단순한 텍스트 변환을 넘어, 날씨와 사용자의 감정을 다차원적으로 분석합니다. 아래 산점도는 사용자의 기분(Emotion)과 생성된 릴레이의 길이(Relay Depth) 사이의 상관관계를 보여줍니다.
