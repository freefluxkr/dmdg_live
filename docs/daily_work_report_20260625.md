# 🍊 일일 작업 보고서 (2026-06-25)

## 📌 오늘 완료한 주요 작업 내역 (아침 피드백 11가지 및 런타임 버그 완벽 대응)

### 1. 프론트엔드 UI/UX 대규모 개편 및 최적화
- **스크롤 레이아웃 앱 네이티브화**: 전체 페이지가 불필요하게 스크롤되는 문제를 해결하기 위해 최대 높이를 화면에 맞추고(`fixed inset-0 overflow-hidden`), 뷰포트 내에서 깔끔하게 정렬되도록 구조를 개선했습니다.
- **오디오 팝업(모달) UI 도입**: 평소에는 화면을 넓게 쓰고, 사용자가 '녹음'을 하거나 '다른 사람의 릴레이'를 재생할 때만 중앙에 예쁜 오디오 팝업이 뜨도록 구현했습니다.
  - 기존의 자리 차지하던 Blob(물방울) 감성 큐레이터 애니메이션을 팝업 안으로 깔끔하게 이동시켰습니다.
  - 녹음 시작 시 `mediaRecorder`와 연결된 `micAnalyser`를 사용하여 사용자의 실제 목소리 파형에 반응하도록 애니메이션 버그를 수정했습니다.
  - 팝업 내부에 크고 직관적인 `[■ 중지]` 버튼을 추가하여 조작성을 높였습니다.

### 2. 다국어 및 번역 기능 최적화
- **일본어 배경음(TTS) 삭제**: 기존 한국어 재생과 겹쳐서 기괴하게 들리던 일본어 합성음 엔진 부분을 과감하게 삭제하여 원본 목소리의 감동에 집중하도록 했습니다.
- **번역 기능 최적화**: 중복으로 자리만 차지하던 '실시간 번역' 버튼을 걷어내고 텍스트 중심의 깔끔한 뷰를 구성했습니다.
- **지원 언어 정상화**: `fr`(프랑스어), `de`(독일어) 선택 시 `i18n` 사전에 데이터가 누락되어 언어가 변경되지 않던 치명적 오류를 해결하기 위해 직접 HTML 딕셔너리 데이터를 파싱해 주입했습니다.

### 3. 런타임 JS 버그 및 캐시 동기화 해결 (트러블슈팅)
- **ReferenceError (TDZ) 및 Null 참조 에러 해결**: 번역 버튼과 기존 물방울 캔버스 태그 제거 이후, 해당 태그를 참조하던 구 자바스크립트들이 연쇄적으로 붕괴하여 앱 UI 작동을 정지시키는 문제를 완벽 차단했습니다.
  - `micAudioCtx` 변수의 호이스팅 및 Temporal Dead Zone(TDZ) 오류를 `let` -> `var`로 교체하여 블록 스코프 문제를 우회했습니다.
  - 삭제된 캔버스 DOM을 참조하는 찌꺼기 코드(`drawBlob`, `canvas.getContext('2d')`)를 모두 발본색원하여 제거했습니다.
- **Service Worker 버전 펌핑**: 오프라인 모드 PWA로 인해 구버전이 지속적으로 노출되던 문제를 `sw.js`의 버전을 `v13`까지 지속적으로 갱신하여 클라이언트의 브라우저에서 강제로 최신판을 바라보도록 멱등성을 확보했습니다.

### 4. Git 보안 룰렛 및 저장소 최적화
- **Github Secret Scanning 회피**: 기존 커밋에 Firebase 비공개 키(`serviceAccountKey.json`, `firebase-adminsdk-api_key.json`)가 포함되어 Github 서버단에서 `Push Protection`으로 튕겨내는 문제를 해결했습니다.
  - `git reset --soft`와 `git rm --cached`를 통해 위험한 커밋 내역을 로컬에서 하나로 병합(Squash) 및 세탁한 뒤 푸시를 진행하여 보안 사고를 선제 방지했습니다.
