* **진행상황 보고**: 현재 진행 중인 작업 내역과 상황 보고는 항상 한국어로 명확하게 알려주세요.
* **솔루션 중심**: 에러나 버그에 대해 불필요한 사과를 남발하기보다, 원인 분석과 명확한 해결책을 우선 제시하세요.

## 🎨 2. 프론트엔드 UI/UX 가이드라인
* **CDNs 및 외부 라이브러리**: Tailwind CSS, Lucide Icons, Dexie.js 등은 함부로 로컬 모듈로 설치하지 말고, 기존의 CDN 로드 구조를 무조건 유지하세요.
* **프리미엄 디자인 테마**: 주황빛과 핑크색이 은은하게 어우러진 그라데이션 및 글라스모피즘(Glassmorphism) 스타일을 손상시키지 마세요.
* **모바일 마이크로 인터랙션**: 버튼과 카드 탭 시에 `:active` 축소 스케일링(`transform: scale(0.95)`)과 `touch-target` 표준(최소 44x44px 이상 터치 영역 보장)을 확실하게 지켜주세요.

## 🎧 3. 핵심 사운드 및 오디오북 제약 규칙 (Strict Rules)
* **iOS Safari (아이폰) TTS 빗장 풀기**:
  * 사용자가 개별 카드나 재생 컨트롤러를 직접 탭(Touch)하는 동기적 시점에 **`unlockTts()`**를 즉시 실행하여 iOS 오디오 세션을 해제해야 합니다.
  * 비동기 함수 안이나 지연 콜백 내부에서 최초 `speechSynthesis.speak()`를 단독 호출해서는 안 됩니다.
* **Promise 기반 순차 재생 제어**:
  * 한국어 원본 음성 재생(`audioEngine.playChunk`)이 다 끝난 후에 일본어 TTS가 동작하도록 체이닝 규칙을 철저히 검증하세요. 두 소리가 겹치거나 엉켜 나와서는 안 됩니다.

## 🕷️ 4. 데이터베이스 및 크롤러 연동
* **증분 적재 (MD5 해시 중복 제거)**:
  * 크롤링 데이터를 Firestore에 밀어 넣을 때, 본문 공백을 제거한 MD5 해시값 대조를 반드시 수행하여 중복 삽입을 원천 차단하세요.
* **명시적 Firebase 인증 정보 전달**:
  * `firebase-admin`과 Firestore 클라이언트를 연동할 때, 기본 ADC(Application Default Credentials)에 꼬이지 않도록 명시적으로 `firestore.client(app=app)` 형식으로 초기화하여 사용해야 합니다.
